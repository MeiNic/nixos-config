# NixOS Configuration

Personal NixOS system configuration for a laptop with an Intel Core Ultra 155H (Meteor Lake), running the **Cinnamon** desktop environment with dual-boot support for CachyOS.

---

## Module Overview

```
configuration.nix          ← Entry point (networking, locale, services, state)
hardware.nix               ← Boot, LUKS/FIDO2, kernel, power management, Bluetooth, firmware
filesystems.nix            ← btrfs subvolume mounts on the shared LUKS partition
desktop.nix                ← Display manager, Cinnamon DE, audio (PipeWire), printing, input
user.nix                   ← User account, packages, Zsh, ADB, Wireshark
git-ssh.nix                ← Git config, SSH key deployment (migrated from Windows)
backup.nix                 ← btrbk snapshots + BorgBackup to USB drives
xdg-defaults.nix           ← Default applications (MIME types, BROWSER/EDITOR vars, xdg-portal)
hardware-configuration.nix ← Auto-generated by nixos-generate-config (do not edit)

secrets/
  personal.nix             ← Git identity & SSH signing key (gitignored, fill from template)
  personal.nix.template    ← Template for personal.nix
  luks-uuids.nix.template  ← Template for sensitive LUKS UUIDs
```

## Daily Workflow

The `@configs` btrfs subvolume is mounted at `~/nixos-config` (owned by `nico`), so all git operations work **without `sudo`**. `/etc/nixos` is a symlink pointing to `~/nixos-config`, which is re-created on every rebuild by an activation script.

```bash
# Edit config files, then commit and push (no sudo needed):
cd ~/nixos-config
git add -A && git commit -m "feat: describe change"
git push

# Apply changes to the running system:
sudo nixos-rebuild switch
# Or use the shell alias:
nixos-rebuild          # alias → sudo nixos-rebuild switch

# Commit + push with a timestamped message in one step:
nixos-sync             # alias → git add -A && git commit -m "chore: sync YYYY-MM-DD" && git push
```

> **Note:** `nico` has a `NOPASSWD` sudo rule specifically for `nixos-rebuild`, so the password prompt only appears for other sudo commands.

### LUKS / Dual-boot Sensitive Values

The CachyOS boot entry in `hardware.nix` contains placeholder values (`<EFI-TOKEN>`, `<CACHYOS-LUKS-UUID>`). Replace these with your own values from `blkid` output. These are intentionally **not** stored in git.

## Filesystems (`filesystems.nix`)

All subvolumes live on the **shared LUKS partition** (`/dev/mapper/shared`, btrfs). All are mounted with `compress=zstd` and `noatime`.

| Mount Point | btrfs Subvolume | Notes |
|---|---|---|
| `/nix` | `@nix` | Nix store |
| `/home` | `@home` | User home directories |
| `/home/nico/nixos-config` | `@configs` | Config files — git-managed, no sudo needed |
| `/var/lib/docker` | `@docker` | Docker data |
| `/var/lib/flatpak` | `@flatpak` | Flatpak apps |

> **Important:** The `/home/nico/nixos-config` mount point is **not** needed for boot (`neededForBoot = false`). VS Code or your file manager may show it as a mountable/unmountable volume — **do not unmount it while the system is running**, as `/etc/nixos` is a symlink that points to it. Unmounting would break the symlink until the next reboot/remount.

### Multi-Monitor Panels

A user systemd service (`cinnamon-panel-setup`) runs on every login and is also triggered automatically by a **udev hotplug rule** whenever a display is connected or disconnected. It uses `xrandr` to detect how many monitors are active and dynamically writes the correct `dconf` panel keys — no manual intervention or rebuild needed.

Each connected monitor gets an identical bottom panel:
- **Left:** App-menu + Window-list
- **Right:** Systray + Calendar

After a rebuild, log out and back in once to activate the new service. From that point on, plugging in any display will automatically apply the panel layout.

## Default Applications (`xdg-defaults.nix`)

Default programs are set through three complementary layers:

| Layer | Mechanism | Scope |
|---|---|---|
| **A** | `environment.sessionVariables` (`BROWSER`, `EDITOR`, `VISUAL`, `TERMINAL`, `PAGER`) | Terminal/scripts, legacy programs |
| **B** | `xdg.mime.defaultApplications` → `/etc/xdg/mimeapps.list` | GUI apps, `xdg-open`, file manager |
| **C** | `xdg.portal` (GTK portal) | Flatpak file dialogs, screen sharing |

User overrides via Cinnamon's **"Open With…"** dialog are written to `~/.config/mimeapps.list` and always take priority over the system defaults.

### Current Defaults

| MIME / purpose | Program |
|---|---|
| Web browser | Brave |
| E-mail | Thunderbird |
| PDF | Evince |
| Office documents | LibreOffice (Writer / Calc / Impress) |
| Images | gThumb |
| Video & Audio | VLC |
| Archives | File Roller |
| Text / Code | VS Code |
| File manager | Nemo (Cinnamon) |

To change a default, edit the relevant entry in `xdg-defaults.nix` and run `sudo nixos-rebuild switch`. To find the correct `.desktop` ID:
```bash
find /run/current-system/sw/share/applications ~/.local/share/applications -name '*.desktop' | sort
# Check what's currently registered for a type:
xdg-mime query default application/pdf
```

## Git & SSH (`git-ssh.nix`)

All personal identifiers (name, email, signing key) are loaded from `secrets/personal.nix` (gitignored). Copy `secrets/personal.nix.template` to `secrets/personal.nix` and fill in your values.

| Feature | Detail |
|---|---|
| Git config | Written declaratively via activation script to `~/.gitconfig` |
| Commit signing | SSH signing with a dedicated `git_signing_key` Ed25519 key |
| GPG agent | Enabled (SSH support disabled — SSH auth handled by ssh-agent) |
| SSH daemon | Password auth disabled, root login disabled |

### Migrating SSH / GPG Keys from Windows

Key files are expected in `~/` after migration:
- `~/gh_action_key` + `~/gh_action_key.pub` — GitHub Actions key
- `~/my_public_key.asc` + `~/my_secret_key.asc` — GPG key (for Thunderbird OpenPGP)
- `~/revoke.asc` — GPG revocation certificate

Import the GPG key for Thunderbird once:
```bash
iconv -f UTF-16LE -t UTF-8 ~/my_public_key.asc | tail -c +4 | tr -d '\r' | gpg --import
iconv -f UTF-16LE -t UTF-8 ~/my_secret_key.asc | tail -c +4 | tr -d '\r' | gpg --import
```

## Backup (`backup.nix`)

A two-layer backup strategy:

### Layer 1 — btrfs Snapshots (local)

- **Tool:** btrbk
- **Location:** `/.snapshots/` (`@snapshots` subvolume)
- **Schedule:** Hourly (keep 24h) + daily at midnight (keep 14d / 8w / 12m)
- **Subvolumes snapshotted:** root, `@home`, `@configs`, `@nix`, `@docker`, `@flatpak`

### Layer 2 — BorgBackup to USB Drive A (automatic)

- **Timer:** daily at 02:30 (`Persistent=true` — catches up after missed runs)
- **Drive absent:** exits cleanly, no failure logged
- **Compression:** `auto,zstd`
- **Checkpointing:** every 5 minutes (interrupted backups resume)

### Layer 3 — BorgBackup to USB Drive B (manual)

- **No timer** — triggered manually only
- ```bash
  sudo systemctl start borgbackup-job-usb-manual
  ```

### What is Backed Up

| Included | Excluded |
|---|---|
| `/home`, `/etc/nixos`, `/var/lib/docker`, `/var/lib/flatpak` | `/.cache`, `__pycache__`, `.npm`, `.cargo/registry`, `.rustup`, Trash, `docker/overlay2` |

### USB Drive Setup (one time per drive)

```bash
# 1. Format and label the partition
sudo mkfs.ext4 -L borg-usb-a /dev/sdX1
sudo mkfs.ext4 -L borg-usb-b /dev/sdX1

# 2. Create mount points
sudo mkdir -p /mnt/borg-usb-a /mnt/borg-usb-b

# 3. Initialise Borg repositories (drive must be plugged in)
export BORG_PASSPHRASE="your-strong-passphrase"
sudo borg init --encryption=repokey-blake2 /mnt/borg-usb-a/borg-repo
sudo borg init --encryption=repokey-blake2 /mnt/borg-usb-b/borg-repo

# 4. Store the passphrase
sudo mkdir -p /etc/nixos/secrets
echo "your-strong-passphrase" | sudo tee /etc/nixos/secrets/borg-passphrase
sudo chmod 600 /etc/nixos/secrets/borg-passphrase
```

### Useful Backup Commands

```bash
# Check last backup status
sudo systemctl status borgbackup-job-usb-a
sudo systemctl status borgbackup-job-usb-manual

# List archives on a drive
sudo BORG_PASSPHRASE=$(cat /etc/nixos/secrets/borg-passphrase) \
  borg list /mnt/borg-usb-a/borg-repo

# Restore a specific file
sudo BORG_PASSPHRASE=$(cat /etc/nixos/secrets/borg-passphrase) \
  borg extract /mnt/borg-usb-a/borg-repo::archive-name path/to/file
```

## Applying Changes

```bash
# Full rebuild and switch
sudo nixos-rebuild switch

# Test a config change without making it the default boot entry
sudo nixos-rebuild test

# Build but don't switch (inspect the result)
sudo nixos-rebuild build
```
